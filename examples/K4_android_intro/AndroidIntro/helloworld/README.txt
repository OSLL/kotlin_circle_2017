AndroidManifest.xml - это файл, содержащий информацию о приложении, которая требуется системе Android:
 задает имя пакета Java для приложения. Это имя пакета служит уникальным идентификатором приложения.
 описывает компоненты приложения — операции, службы, приемники широковещательных сообщений и поставщиков контента, из которых состоит приложение.
 содержит имена классов, которые реализуют каждый компонент, и публикует их возможности (указывает, например, какие сообщения Intent они могут принимать). На основании этих деклараций система Android может определить, из каких компонентов состоит приложение и при каких условиях их можно запускать.
 определяет, в каких процессах будут размещаться компоненты приложения.
 объявляет, какие разрешения должны быть выданы приложению, чтобы оно могло получить доступ к защищенным частям API-интерфейса и взаимодействовать с другими приложениями.
 также объявляет разрешения, требуемые для взаимодействия с компонентами данного приложения.
 содержит список классов Instrumentation, которые при выполнении приложения предоставляют сведения о профиле и прочую информацию. Эти объявления присутствуют в файле манифеста только во время разработки и отладки приложения и удаляются перед его публикацией.
 объявляет минимальный уровень API-интерфейса Android, который требуется приложению.
 содержит список библиотек, с которыми должно быть связано приложение.

<application> - декларация приложения. Содержит описание всех компонентов приложения и их атрибуты.


activity_main.xml - это layout-файл, в нем определяется набор и расположение компонентов, которые будут видны на экране.
Файл используется в 3-х режимах представления: Text, Design и Blueprint (Desing + Blueprint). Blueprint - только контуры View-компонентов.
Palette - это список View компонентов, которые могут быть добавлены на экран.
Component tree - это иерархия View компонентов экрана.
В текстовом режиме, названия xml-элементов - это классы View-элементов, xml-атрибуты - это параметры View-элементов, т.е. все параметры, которые можно изменить в Properties.
Можно вносить изменения в текстовом режиме, они также будут отображаться в Design.
Можно использовать разные layout для вертикальной/горизонтальной ориентации для одной Activity.
